## 概要

方式B（連続撮影＋ページ結合）において、OCR結果のページ間重複を**決定論的に検出して結合**するアルゴリズムと、**疑義検知ルール**（Haiku再OCRトリガ）の仕様。LLMはOCRと疑義ページの再OCRに限定し、結合判定はローカル処理で行う。

## 設計原則

- **LLMにはOCRのみやらせる**。結合判定は決定論的アルゴリズムで行う
- **RapidFuzz（近似文字列一致）** で末尾N行×先頭M行の重複を検出
- **コードブロック内は空白正規化を弱める**（インデントが意味を持つ）
- **正規化は類似度計算時のみ**適用し、**保存・出力されるテキストは正規化前のオリジナルを保持する**（NFKC等でMarkdown本文が意図せず変化するのを防ぐ）
- **in_codeマスクはページ全体で生成してからスライスする**（tail/headスライスの開始位置がコードブロック途中の場合にマスクが破綻しないため）
- 結合できなかった場合は `<!-- MERGE_UNCERTAIN -->` を注釈として残し、後段（再OCR／手直し）に回す

## パラメータ

| 名前 | 既定値 | 役割 |
|------|--------|------|
| `TAIL_LINES` | 40 | 前ページの末尾から何行取って重複検出するか |
| `HEAD_LINES` | 60 | 次ページの先頭から何行取って重複検出するか |
| `MAX_SHIFT` | 3 | 窓のずれを何行まで許容するか（先頭の空行やゴミ行を吸収） |
| `MIN_SCORE` | **91.0** | RapidFuzzのスコア閾値（0-100）。**初期値を90-92推奨**（OCR誤認1-2文字で88を下回りやすいため）。境界欠落が起きるなら下げる |
| `MIN_SAFE_K` | 3 | これ未満の短い重複は原則採用しない（空行や記号の偶発マッチ回避、多様性条件でのみ採用） |
| `PREFER_LONGER_WITHIN` | 1.5 | 同点時は長いk（重複長）を優先 |
| `MAX_PAGES` | 20 | 結合ループの最大ページ数（要件 F-11） |

## 擬似コード（Python）

```python
"""
Minimal page-merge algorithm (deterministic) for OCR'ed Markdown pages.

Dependencies:
  - rapidfuzz (for similarity)
Optional:
  - skimage.metrics (SSIM) or cv2 for image similarity assist
"""

from dataclasses import dataclass
import re
import unicodedata
from typing import List, Tuple, Optional, Dict

TAIL_LINES = 40
HEAD_LINES = 60
MAX_SHIFT = 3
MIN_K = 1
MIN_SAFE_K = 3              # これ未満の短い重複は多様性条件付きでのみ採用
MIN_SCORE = 91.0            # 初期値を引き上げ（OCR誤認1-2文字で88を下回りやすい）
PREFER_LONGER_WITHIN = 1.5
MAX_PAGES = 20              # 結合ループ上限（要件 F-11）

FENCE_RE = re.compile(r"^\s*```")
# ignore行判定用: 空行、記号のみ、極端に短い行（ページ番号やステータスバー相当）
IGNORE_LINE_RE = re.compile(r"^\s*$|^[\W_]{1,}$|^\s*\d{1,3}\s*$")
# 多様性判定用: 英数字を含むか
HAS_ALNUM_RE = re.compile(r"[A-Za-z0-9぀-ヿ一-鿿]")

@dataclass
class OverlapDecision:
    ok: bool
    k: int
    score: float
    shift_prev: int
    shift_next: int
    reason: str

def split_lines(md: str) -> List[str]:
    return md.replace("\r\n", "\n").replace("\r", "\n").split("\n")

def compute_in_code_mask(lines: List[str]) -> List[bool]:
    in_code = False
    mask = []
    for line in lines:
        if FENCE_RE.match(line):
            mask.append(in_code)
            in_code = not in_code
            continue
        mask.append(in_code)
    return mask

def normalize_for_compare(line: str, in_code: bool) -> str:
    """類似度計算 **専用** の正規化。保存テキストには使わない。"""
    line = line.replace("​", "")  # zero-width space
    line = line.rstrip("\n")
    if in_code:
        # コード行: タブをスペースに展開、末尾空白除去のみ（インデント意味温存）
        line = line.replace("\t", "    ")
        return line.rstrip()
    else:
        # 本文: NFKC正規化＋引用符・ダッシュの種別吸収＋空白圧縮
        s = unicodedata.normalize("NFKC", line)
        s = s.replace("“", '"').replace("”", '"').replace("’", "'")
        s = s.replace("—", "-").replace("–", "-").replace("−", "-")
        s = re.sub(r"[ \t]+", " ", s).strip()
        return s

def join_for_compare(lines: List[str], mask: List[bool]) -> str:
    return "\n".join(normalize_for_compare(ln, c) for ln, c in zip(lines, mask))

def is_ignorable(line: str) -> bool:
    """空行・記号のみ・ページ番号等のゴミ行判定（MAX_SHIFT吸収対象の拡張）"""
    return bool(IGNORE_LINE_RE.match(line))

def has_diversity(lines: List[str], min_alnum_lines: int = 2) -> bool:
    """重複区間に英数字を含む行が min_alnum_lines 以上含まれるか（偶発マッチ防止）"""
    return sum(1 for ln in lines if HAS_ALNUM_RE.search(ln)) >= min_alnum_lines

def similarity(a: str, b: str) -> float:
    from rapidfuzz import fuzz
    return float(fuzz.ratio(a, b))

def best_overlap(prev_md: str, next_md: str,
                 tail_lines: int = TAIL_LINES,
                 head_lines: int = HEAD_LINES,
                 max_shift: int = MAX_SHIFT) -> OverlapDecision:
    prev_lines = split_lines(prev_md)
    next_lines = split_lines(next_md)

    # in_codeマスクはページ全体で生成してからスライスする。
    # tail/headスライスの開始位置がコードブロック途中の場合にマスクが破綻しないため。
    prev_mask_full = compute_in_code_mask(prev_lines)
    next_mask_full = compute_in_code_mask(next_lines)

    tail_start = max(0, len(prev_lines) - tail_lines)
    tail = prev_lines[tail_start:]
    tail_mask = prev_mask_full[tail_start:]
    head = next_lines[:head_lines]
    head_mask = next_mask_full[:head_lines]

    def max_ignorable_prefix(xs, limit):
        """空行 OR 記号のみ OR 極端に短い行（is_ignorable）が先頭何行続くか"""
        c = 0
        for i in range(min(len(xs), limit)):
            if is_ignorable(xs[i]):
                c += 1
            else:
                break
        return c

    ignorable_prefix = max_ignorable_prefix(head, max_shift)
    best: Optional[OverlapDecision] = None

    for i in range(0, min(max_shift, len(tail)) + 1):
        for j in range(0, min(max_shift, len(head)) + 1):
            if j > ignorable_prefix:
                continue
            tail2 = tail[i:]
            head2 = head[j:]
            tail_mask2 = tail_mask[i:]
            head_mask2 = head_mask[j:]
            max_k = min(len(tail2), len(head2))
            if max_k < MIN_K:
                continue
            for k in range(max_k, MIN_K - 1, -1):
                a = join_for_compare(tail2[-k:], tail_mask2[-k:])
                b = join_for_compare(head2[:k], head_mask2[:k])
                sc = similarity(a, b)
                if sc < MIN_SCORE:
                    continue

                # 小k抑制: k < MIN_SAFE_K の場合は多様性条件を満たすときのみ採用
                if k < MIN_SAFE_K:
                    if not has_diversity(head2[:k]):
                        continue

                cand = OverlapDecision(
                    ok=True, k=(j + k), score=sc,
                    shift_prev=i, shift_next=j,
                    reason=f"matched k={k} (drop next first {j}+{k} lines)"
                )
                if best is None:
                    best = cand
                else:
                    if (cand.score > best.score + PREFER_LONGER_WITHIN) or \
                       (abs(cand.score - best.score) <= PREFER_LONGER_WITHIN and cand.k > best.k):
                        best = cand
                break
    if best is None:
        return OverlapDecision(
            ok=False, k=0, score=0.0, shift_prev=0, shift_next=0,
            reason="no overlap above threshold; boundary uncertain"
        )
    return best

def merge_pages(prev_md: str, next_md: str, decision: OverlapDecision) -> Tuple[str, Dict]:
    """
    重要: 結合に使う行データは **オリジナルの prev_md / next_md** であり、
    normalize_for_compare による正規化済み文字列は類似度計算にのみ使用する。
    保存・出力されるテキストにはNFKC等の変換は**適用しない**。
    """
    prev_lines = split_lines(prev_md)
    next_lines = split_lines(next_md)
    if not decision.ok or decision.k <= 0:
        merged = prev_md.rstrip() + "\n\n<!-- MERGE_UNCERTAIN: no reliable overlap -->\n\n" + next_md.lstrip()
        return merged, {"merge_uncertain": True, "overlap": decision.__dict__}
    kept_next = next_lines[decision.k:]
    merged = "\n".join(prev_lines + kept_next)
    return merged, {"merge_uncertain": False, "overlap": decision.__dict__}
```

## SSIM画像類似度の補助利用（オプション）

- 前ページ下部スライスと次ページ上部スライスのSSIMを計算
- **SSIMが高い（画像的には重なっている）のにテキスト重複スコアが低い → OCRノイズが疑われる**
  → 該当ページをHaiku再OCRトリガに
- **直近2枚の画像全体がSSIM閾値超で連続 → スクロール終端（同一画面）と判定**
  → `---END---`マーカー欠落のフェイルセーフとして終了判定に使う

## 疑義検知ルール

ページごとに以下のいずれかに該当したら「疑義あり」→ Haiku再OCRまたは再撮影。

### 1. コードブロック内のインデント揺れ
- タブとスペースが混在（`\t`を含む行あり）
- 先頭インデント幅が2/4の倍数に寄らない行が多い
- 同一ブロック内で`^\s+`の長さがランダムに増減

### 2. 括弧・引用符・バッククォートの不整合
- `()` `[]` `{}` の個数差が2以上
- `"` または `'` が極端に奇数（コード優先で判定）
- インラインコード `` ` `` が奇数
- code fence ``` の出現回数が奇数（ブロックが閉じていない）

### 3. 行末欠落の兆候（VSCode撮影で出やすい）
- 行末が `,` `.` `:` `;` `=` `==` `->` `=>` で終わる行が全行の10%超
- 行末が英数字1文字で終わる行が連続
- URLが `http` で途切れる、`import` / `from` が途中で終わる等の典型パターン

### 4. `---END---`マーカー欠落（期待ページ）
- ユーザーが最終ページと指示したが`---END---`が無い
- 最大ページ上限に到達したのに`---END---`が無い
- 対応: **まず再撮影**（OCRより画質が原因のことが多い）

### 5. 結合信頼度の低さ（ページ間）
- `best_overlap()`が`ok=False`
- `score < MIN_SCORE + 2` のギリギリ
- `k`が1〜2行しか取れない（`MIN_SAFE_K`未満で多様性条件も満たさない場合は不採用）
- 対応:
  - **境界の2ページだけHaiku再OCR**が費用対効果が高い
  - スラッシュコマンド側で**境界画像（前ページ末尾＋次ページ先頭）を並列表示**し、ユーザー手修正を支援（要件 F-13）

### 6. OCR由来のゴミ文字
- `` U+FFFD（replacement char）、意味不明な罫線、連続する同一記号等

## 運用チューニング

- `MIN_SCORE` は **91** で開始。重複残存が目立つなら下げる（重複を積極的に検出する方向）、境界欠落が起きるなら上げる
- `MIN_SAFE_K` は **3** で開始。短い重複（k=1-2）の偶発マッチが多ければ4-5に上げる
- 実サンプル（2〜3ページのOCR結果）が取れたら、パラメータを実測ベースで再調整する
- **`TAIL_LINES` / `HEAD_LINES` の動的調整（将来）**: 最初の1-2ページのOCR行数から1画面あたりの実行数を推定し、それに応じて探索範囲を縮小（性能改善・偶発マッチ抑制）

## 撮影前処理

- OCR前にラプラシアン分散でブレ検知、閾値（**初期値: 100**）未達は撮り直し要求
- **撮影間隔クールダウン**（500ms程度）を挟み、カメラのAE/AF安定を待つ
- **EXIFメタデータを除去**（`exiftool -all=` 等）してからLLMに送信

> **MVPでリサイズのみを実施する。** 4MP（2688×1520px）原寸送信ではGemini Flashのトークンコストが非機能要件（1ページ < 0.5円）を超える可能性があるため、**長辺1600-2000pxへのリサイズをコスト安全装置として導入する**（Pillow数行）。
> シャープ・JPEG品質調整はMVP外。シャープはClearType環境で虹縁を強調してOCR誤認を増やす副作用があるため、精度問題が実測で確認されてから判断する。
